Additional phase-discipline rules:

- At the end of each phase, summarize the canonical outputs created in that phase.
- Treat those outputs as locked inputs for the next phase unless explicitly revised.
- If a later phase reveals a problem in an earlier phase, do not silently change it; instead, report the mismatch clearly and propose a revision.
- Never rename a field, enum, or approval state after it has been defined unless explicitly instructed.
